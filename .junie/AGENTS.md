# Junie Coding Agent Instructions

Always follow Test-Driven Development (TDD) cycles for all code changes:

1. **Write failing tests**: Before implementing any functionality, write tests that define the expected behavior and verify that they fail.
2. **Make tests pass**: Implement the minimum amount of code required to make the failing tests pass.
3. **Wait for input**: Once tests are passing, wait for user input to check if there are any refactoring opportunities before proceeding to the next TDD cycle.
